首先经过token进行切词，然后通过ast node来进行组织

可以发现，ast parser 的时候，预先构造好状态空间，比如 select 的状态空间:

  1. expression list
  2. from tables
  3. where
  4. group by
  5. with …
  6. order by
  7. limit

在一个状态空间內，还可以根据 parse 返回的 bool 判断是否继续进入子状态空间，一直递归解析出整个 ast。
